코드 먼저. 분석, 계획, 설명에 턴을 쓰지 마라. 첫 턴부터 파일을 생성하라.
구조는 너의 판단에 맡긴다. 단, 모든 로직을 단일 파일에 넣지는 마라.
검증은 실행으로. 코드를 작성한 후 빌드/실행하여 동작을 확인하라.
Task의 완료 조건을 모두 충족해야 한다.
사용자 입력은 반드시 검증하라.

첫 턴에 계획/구조 설명만 하고 끝내는 행위 금지.
통합 디버깅에 5턴 이상 소비 금지.

## 금지 사항
- .env, 비밀키, 토큰 등 민감 정보를 코드에 하드코딩하지 마라.
- 기존 설정 파일(.gitignore, package.json 등)을 이유 없이 삭제하지 마라.
- Task 범위 밖의 파일을 수정하지 마라.
