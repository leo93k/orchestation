# UI 디자인 원칙 — OpenAI Assistants Playground 분석

> 참고 UI: OpenAI Assistants API 설정 화면
> 목적: 이 디자인의 특징을 분석하여 우리 프로젝트 UI에 적용

---

## 1. 다크 테마 기반

- **배경**: 순수 블랙(#000)이 아닌 짙은 그레이(약 #1a1a1a ~ #202020)
- **카드/입력 영역**: 배경보다 약간 밝은 그레이(약 #2a2a2a ~ #333)로 레이어 구분
- **텍스트**: 순백(#fff)이 아닌 약간 톤다운된 화이트(약 #e5e5e5)
- **효과**: 눈의 피로 감소, 요소 간 자연스러운 깊이감(depth) 형성

## 2. 섹션 라벨링

- **대문자 라벨**: `TOOLS`, `MODEL CONFIGURATION` 등 섹션 제목이 **소형 대문자(uppercase) + 작은 폰트 + 연한 회색**
- **일반 라벨**: `Name`, `System instructions`, `Model` 등 필드 라벨은 **일반 크기 + 흰색**
- **효과**: 시각적 위계(hierarchy)가 명확 — 섹션 > 필드 > 값 순서로 읽힘

## 3. 입력 필드 스타일

- **보더**: 얇은 1px 보더, 배경보다 약간 밝은 색 (#3a3a3a 정도)
- **라운드**: 약 6~8px border-radius (과하지 않은 둥근 모서리)
- **내부 패딩**: 넉넉한 padding (약 12~16px)
- **포커스 없는 상태**: 보더가 거의 배경에 녹아듦
- **효과**: 입력 영역이 존재하지만 시각적으로 "튀지" 않음

## 4. 토글 & 컨트롤

- **토글 스위치**: 작고 미니멀한 원형 토글, 비활성 시 어두운 색
- **슬라이더**: 얇은 트랙 + 작은 원형 핸들, 활성 구간만 하이라이트 컬러
- **드롭다운**: 입력 필드와 동일한 스타일 + 우측 화살표 아이콘
- **효과**: 컨트롤 종류가 달라도 시각적 무게가 균일

## 5. 액션 버튼

- **`+ Files`, `+ Functions`**: 우측 정렬, 작은 크기, 테두리 스타일(ghost button)
- **색상**: 배경과 비슷한 톤, 호버 시에만 강조
- **효과**: 주요 콘텐츠를 방해하지 않으면서 액션 접근성 유지

## 6. 레이아웃 원칙

- **단일 컬럼**: 설정 항목이 세로로 나열 — 좌우 분산 없음
- **일정한 간격**: 섹션 간 약 24~32px, 필드 간 약 16~20px
- **좌측 정렬 통일**: 라벨, 입력, 토글 모두 좌측 정렬 기준선 공유
- **우측에 부가 액션**: 토글 행의 오른쪽에 설정 아이콘, + 버튼 배치
- **효과**: 시선이 위에서 아래로 자연스럽게 흐름 (F-패턴)

## 7. 타이포그래피

- **폰트**: 산세리프, 일정한 자간
- **크기 단계**: 최소 3단계 (섹션 라벨 ~11px / 필드 라벨 ~14px / 입력값 ~14px)
- **서브텍스트**: ID 등 보조 정보는 작은 크기 + 연한 회색 (#888)
- **효과**: 정보 우선순위가 글자 크기만으로 전달됨

## 8. 정보 밀도

- **최소 장식**: 아이콘은 ⓘ(정보), ⚙(설정) 등 기능적 용도만
- **여백 활용**: 요소 사이에 충분한 breathing room
- **그룹핑**: 관련 항목끼리 묶되, 구분선 대신 간격(spacing)으로 분리
- **효과**: 많은 설정을 담고 있지만 복잡해 보이지 않음

## 요약: 적용 체크리스트

| 원칙 | 키워드 |
|------|--------|
| 다크 테마 | 3단계 그레이 레이어 (배경 < 카드 < 보더) |
| 위계 | 대문자 섹션 라벨 + 일반 필드 라벨 |
| 입력 필드 | 얇은 보더, 넉넉한 패딩, 녹아드는 스타일 |
| 컨트롤 | 균일한 시각적 무게의 토글/슬라이더/드롭다운 |
| 버튼 | Ghost 스타일, 우측 정렬, 작은 크기 |
| 레이아웃 | 단일 컬럼, 좌측 정렬, F-패턴 |
| 타이포 | 3단계 크기, 보조 텍스트는 회색 |
| 밀도 | 간격으로 그룹핑, 장식 최소화 |
