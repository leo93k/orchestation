# PRD: 오케스트레이션 시뮬레이션 테스트

## 목표

선형(순차적) 업무 흐름을 여러 개의 태스크로 나누어 오케스트레이션 파이프라인을 시뮬레이션한다.
실제 서비스 개발이 아닌, 태스크 정의 → 실행 → 리뷰 → 완료의 흐름이 정상 동작하는지 검증하는 것이 목적이다.

## 시뮬레이션 방식

- 각 태스크는 **텍스트 문서 작성**으로 구성 (코드 작성 없음)
- 태스크 간 선형 의존성을 두어 순차 실행을 테스트
- 완료 조건은 파일 존재 여부 및 내용 확인으로 단순화

## 태스크 구성

| ID       | 내용                  | 의존성   |
| -------- | --------------------- | -------- |
| TASK-001 | 프로젝트 개요 문서 작성 | 없음     |
| TASK-002 | 용어 정의 문서 작성     | TASK-001 |
| TASK-003 | 최종 요약 문서 작성     | TASK-002 |

## 검증 포인트

- 태스크가 순서대로 실행되는가
- 선행 태스크가 완료되어야 다음 태스크가 시작되는가
- 리뷰 프로세스가 정상 동작하는가
- 완료 후 상태가 올바르게 업데이트되는가
