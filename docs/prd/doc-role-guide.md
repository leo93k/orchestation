# 역할 프롬프트 가이드

## 설계 원칙

역할 프롬프트는 `docs/roles/` 에 저장되며, `run-task.sh`에서 `--system-prompt`로 Claude에 전달된다.

### DO

- **행동 지침 중심** — "코드 먼저", "검증은 실행으로"
- **금지 사항 명시** — 민감정보 하드코딩 금지, 범위 밖 수정 금지
- **검증 기준 구체화** — 체크리스트, 판정 기준

### DON'T

- **역할명 부여** — "너는 ~전문가다" → 편향 발생
- **전문 분야 나열** — Claude가 이미 아는 것을 지시하면 역효과
- **성격 묘사** — "까탈스러운", "편집증적" → 과잉 행동

## 현재 역할 파일

### general.md (기본)

```
코드 먼저. 첫 턴부터 파일을 생성하라.
구조는 너의 판단에 맡긴다. 단일 파일 금지.
검증은 실행으로. 빌드/실행하여 동작 확인.
사용자 입력은 반드시 검증하라.

금지: 민감정보 하드코딩, 범위 밖 수정
```

### reviewer-general.md (기본 리뷰어)

```
완료 조건 기준으로 검증.
하나씩 대조, 실행 확인.
충족 → "승인", 미충족 → "수정요청" + 사유
```

### reviewer-security.md (보안 리뷰어)

```
완료 조건 + 보안 동시 검증.
SQL Injection, XSS, 하드코딩 비밀키 체크.
보안 이슈 → 심각도 표기 + 수정요청.
```

## Task에서 역할 지정

```yaml
---
role: general          # 작업자 역할
reviewer_role: reviewer-general  # 리뷰어 역할
---
```
