# 프롬프트 성능 비교 실험 결론

> 2개 PRD(URL 단축 서비스, 실시간 채팅앱) × 11개 프롬프트 × 14회+ 실행

## 핵심 발견

### "잘 구현해줘"가 최강

- p2("잘 구현해줘")가 코드 품질 평가에서 **14회 중 7회 1등** (평균 50/60)
- 정교한 프롬프트들이 단 세 글자에 졌다
- Claude의 기본 판단을 방해하지 않는 것이 최선

### PRD > 프롬프트

- PRD에 "PORT 환경변수 필수"를 한 줄 추가 → 거의 모든 프롬프트가 지킴
- 프롬프트에 "보안 검증하라"를 써도 → 여전히 취약점 존재

### 단어 하나의 영향

| 프롬프트 | 비용 | 점수 |
|---------|-----|------|
| "**잘** 구현해줘" | $0.39 | **50** |
| "**간결하게** 구현해줘" | $0.22 | **32** |

### 역할 지정 = 편향

- "실용주의 개발자" → 보안 무시
- "PM + 아키텍트" → 구조 과잉, CSS 깨짐
- 역할명 빼면 품질 향상 (42점 → 48점)

### 검증 루프 = 품질

- 0회 검증: 35점
- 1회 검증: 48점
- 3~4회 검증: 50점

### 병렬 오케스트레이션은 이 규모에서 역효과

- 분해 + 통합 비용 > 병렬 이득
- 캐시가 에이전트마다 별도 → 재활용 불가

## 실용적 가이드

- **PRD에 투자하라** — 프롬프트 다듬는 시간에 PRD를 다듬어라
- **역할명을 쓰지 마라** — Claude의 기본 판단을 방해한다
- **"어떻게"가 아니라 "뭘"을 알려줘라**
