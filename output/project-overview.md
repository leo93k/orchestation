# 프로젝트 개요

## 프로젝트 이름

오케스트레이션 시뮬레이션 테스트

## 프로젝트 목적

선형(순차적) 업무 흐름을 여러 개의 태스크로 나누어 오케스트레이션 파이프라인을 시뮬레이션한다.
실제 서비스 개발이 아닌, **태스크 정의 → 실행 → 리뷰 → 완료**의 흐름이 정상 동작하는지 검증하는 것이 목적이다.

## 참여자 역할

### 감독관 (Orchestrator)

- 전체 태스크 흐름을 관리하고 조율한다
- 태스크 간 의존성을 확인하고, 선행 태스크 완료 후 다음 태스크를 할당한다
- 작업 결과를 리뷰하고, 완료 조건 충족 여부를 판단한다
- 태스크 상태를 업데이트한다

### 작업자 (Worker)

- 할당된 태스크의 완료 조건을 충족하도록 작업을 수행한다
- 지정된 Worktree 안에서만 코드를 수정한다
- main 브랜치를 직접 수정하지 않는다
- 작업 완료 후 변경사항을 커밋한다
